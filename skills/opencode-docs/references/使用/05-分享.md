# 分享

分享您的 OpenCode 对话。

OpenCode 的分享功能允许您创建指向 OpenCode 对话的公开链接，方便与团队成员协作或向他人寻求帮助。

* * *

## [工作原理](05-分享.md#%E5%B7%A5%E4%BD%9C%E5%8E%9F%E7%90%86)

当您分享一段对话时，OpenCode 会：

1. 为您的会话创建一个唯一的公开 URL
2. 将您的对话历史同步到我们的服务器
3. 通过可分享的链接使对话可访问 — `opncd.ai/s/<share-id>`

* * *

## [分享模式](05-分享.md#%E5%88%86%E4%BA%AB%E6%A8%A1%E5%BC%8F)

OpenCode 支持三种分享模式，用于控制对话的共享方式：

* * *

### [手动模式（默认）](05-分享.md#%E6%89%8B%E5%8A%A8%E6%A8%A1%E5%BC%8F%E9%BB%98%E8%AE%A4)

默认情况下，OpenCode 使用手动分享模式。会话不会自动共享，但您可以使用 `/share` 命令手动分享：

```
/share
```

这将生成一个唯一的 URL 并复制到您的剪贴板。

要在 [配置文件](../02-配置.md) 中显式设置手动模式：

```
{

  "$schema": "https://opencode.ai/config.json",

  "share": "manual"

}
```

* * *

### [自动分享](05-分享.md#%E8%87%AA%E5%8A%A8%E5%88%86%E4%BA%AB)

您可以在 [配置文件](../02-配置.md) 中将 `share` 选项设置为 `"auto"`，为所有新对话启用自动分享：

```
{

  "$schema": "https://opencode.ai/config.json",

  "share": "auto"

}
```

启用自动分享后，每个新对话都会自动共享并生成链接。

* * *

### [禁用](05-分享.md#%E7%A6%81%E7%94%A8)

您可以在 [配置文件](../02-配置.md) 中将 `share` 选项设置为 `"disabled"`，完全禁用分享功能：

```
{

  "$schema": "https://opencode.ai/config.json",

  "share": "disabled"

}
```

要在团队中对特定项目强制执行此设置，请将其添加到项目的 `opencode.json` 文件中并提交到 Git。

* * *

## [取消分享](05-分享.md#%E5%8F%96%E6%B6%88%E5%88%86%E4%BA%AB)

要停止分享对话并将其从公开访问中移除：

```
/unshare
```

这将移除分享链接并删除与该对话相关的数据。

* * *

## [隐私](05-分享.md#%E9%9A%90%E7%A7%81)

分享对话时需要注意以下几点。

* * *

### [数据留存](05-分享.md#%E6%95%B0%E6%8D%AE%E7%95%99%E5%AD%98)

共享的对话在您明确取消分享之前将一直保持可访问状态。这包括：

- 完整的对话历史
- 所有消息和回复
- 会话元数据

* * *

### [建议](05-分享.md#%E5%BB%BA%E8%AE%AE)

- 仅分享不包含敏感信息的对话。
- 分享前请检查对话内容。
- 协作完成后请取消分享。
- 避免分享包含专有代码或机密数据的对话。
- 对于敏感项目，请完全禁用分享功能。

* * *

## [企业版](05-分享.md#%E4%BC%81%E4%B8%9A%E7%89%88)

对于企业部署，分享功能可以：

- 出于安全合规考虑 **完全禁用**
- **限制** 为仅通过 SSO 身份验证的用户可用
- **自托管** 在您自己的基础设施上

[了解更多](../05-企业版.md) 关于在您的组织中使用 OpenCode 的信息。
